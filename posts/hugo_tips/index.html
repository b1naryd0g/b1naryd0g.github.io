Hugo

ビルドしても public フォルダに記事が出力されません

記事ファイルの draft を false に変更してください。

draft: true
↓
draft: false

記事の名前を間違えてしまいました。記事を削除するにはどうしたらいいですか

記事ファイルを削除する、もしくは正しい名前にリネームしてください。もし、ビルド済みの場合は記事ファイルを削除してから再ビルドを実行します。

要約 (Read more) を自分で区切りたい

<!--more--> を記事中の区切りたい場所に書きます。

Content Summaries
https://gohugo.io/content-management/summaries/

別の記事にリンクしたい

以下のような形式でリンク URL を作れます。

  • {{< ref "/posts/my-post.md" >}}
  • {{< ref "#title" >}}

Markdown の場合は以下のように使います。

  • [リンク名]({{< ref "/posts/my-post.md" >}})

Links and Cross References:
https://gohugo.io/content-management/cross-references/

(Ananke) Google など検索エンジンにインデックスされない

NOINDEX, NOFOLLOW になっている場合 この手順で解決すると思います。

(Ananke) Google Analytics が有効にならない

HUGO_ENV=production hugo でビルドしてください。

Ananke: A theme for Hugo Sites:
https://themes.gohugo.io/gohugo-theme-ananke/

Production To run in production (e.g. to have Google Analytics show up), run HUGO_ENV=production before your build command. For example: HUGO_ENV=production hugo

GitHub Pages

User and organization sites タイプで URL にアクセスすると 404 が表示されます

  • URL が正しいか確認してください
  • Push から 20 分後にリトライしてみてください
  • リポジトリ名が <username>.github.io になっているか確認してください
  • リポジトリが Public になっているか確認してください
    • https://<username>.github.io/index.html にアクセスしてみてください
      • index.html が表示できるときは、index.html を更新して Push してみてください
